### 步驟二之二：設定 Firestore 安全規則（自動化）

請 Codex 在使用者的專案資料夾建立三個檔案：

**1. `firestore.rules`**：
```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /wordcloud_words/{document} {
      allow read, write: if true;
    }
    match /{document=**} {
      allow read, write: if false;
    }
  }
}
```

**2. `firebase.json`**：
```json
{
  "firestore": {
    "rules": "firestore.rules"
  }
}
```

**3. `.firebaserc`**：
```json
{
  "projects": {
    "default": "[使用者的 Firebase 專案 ID]"
  }
}
```

部署規則：
```powershell
npx.cmd -y firebase-tools@latest deploy --only firestore:rules
```

成功會看到 `Deploy complete!`。

> 💡 之後新工具：編輯 `firestore.rules` 加 `match /votes/{document} { allow read, write: if true; }` → 對 Codex 說「部署 Firestore 規則」 → Codex 自動 `firebase deploy --only firestore:rules`。

## 階段二：連接 MCP

### 步驟三：把 Firebase 註冊為 Codex 的 MCP server

> ✅ 三種 Codex 共用 `~/.codex/config.toml`。

**方法 A：Codex Desktop GUI（推薦）**

1. Desktop → 設定 → **Integrations & MCP** → **Add server**
2. 填：
   - Name：`firebase`
   - Command：Windows 建議填 `npx.cmd`；macOS / Linux 填 `npx`
   - Args：`-y`、`firebase-tools@latest`、`mcp`
3. 儲存

**方法 B：手動編輯 `~/.codex/config.toml`**

```toml
[mcp_servers.firebase]
command = "npx.cmd"
args = ["-y", "firebase-tools@latest", "mcp"]
startup_timeout_sec = 60
tool_timeout_sec = 120
```

> Windows 建議用 `npx.cmd`，比較不會踩到 PowerShell `.ps1` 執行原則問題。macOS / Linux 可用 `command = "npx"`。

**方法 C：CLI**

```bash
codex mcp add firebase -- npx -y firebase-tools@latest mcp
```

> 2026-04-29 實測：本機 Codex Desktop 可用「方法 B」成功載入。若 `codex` CLI 在 Windows 顯示 `Access is denied`，不要卡在 CLI，改用 GUI 或手動改 `config.toml`。

## 如果失敗

對 Codex 說：「Firebase 懶人包失敗，幫我檢查重新處理。」

完全重置：
- **Desktop**：設定 → Integrations & MCP → 刪 `firebase`
- **手動**：編輯 `~/.codex/config.toml` 刪 `[mcp_servers.firebase]` 段
- **CLI**：`codex mcp remove firebase`

重登 Firebase：
```powershell
npx.cmd -y firebase-tools@latest logout
npx.cmd -y firebase-tools@latest login
```

## 常見問題

| 問題 | 原因 | 解法 |
|------|------|------|
| `npx: command not found` | Node.js 尚未安裝或終端機尚未重新載入 PATH | 確認 Node.js 已裝，重啟終端機 |
| PowerShell 顯示 `npx.ps1 cannot be loaded because running scripts is disabled` | Windows 執行原則擋到 `npx.ps1` | 改用 `npx.cmd --version`、`npx.cmd -y firebase-tools@latest ...` |
| Codex 讀 Google Drive 專案資料夾出現 sandbox 權限錯誤 | Codex 需要使用者授權讀寫該路徑 | 同意授權後重跑；不要先判定檔案壞掉 |
| 查詢失敗 | Firebase CLI 尚未登入或登入失效 | 確認 `firebase login` 已成功 |
| `firestore_query_collection` 回 `read_time cannot be in the future` | MCP 查詢工具可能送出未來時間參數 | 先用 `firestore_get_document` 或 `firestore_list_documents` 驗證可讀取，必要時重啟 MCP 後再查 |
| 看不到 Firestore 資料 | Firestore 尚未啟用或資料庫尚未建立 | 確認 Console 已啟用 Firestore |
| `projects:list` 是空的 | 目前登入的 Google 帳號沒有 Firebase 專案權限 | 到 Firebase / Google Cloud IAM 把該帳號加進專案成員 |
| 專案名稱看得到但切換失敗 | 可能用了顯示名稱而不是 Project ID | 到 Firebase 專案設定查 Project ID，例如 `teacherstudy-109ef-d92e1` |
| `Cloud Firestore API has not been used ... or it is disabled` | Firestore API 尚未啟用 | 到 Google Cloud Console 啟用 Cloud Firestore API，等幾分鐘再重試 |
| `firestore:databases:list` 回 `result: []` | API 已啟用，但還沒建立 Firestore database | 建立 `(default)` database，建議位置 `asia-east1` |
| Firebase MCP 沒有 `firestore_add_document` 等工具 | MCP 版本或載入狀態不同 | 先用 Firebase CLI / REST API / SDK 驗證；必要時重啟 Codex Desktop |
| 安全性規則過期 | 用了測試模式 | 改白名單規則（步驟二之二） |
| `Permission denied` | rules 沒允許這個集合 | 檢查 `firestore.rules` 並重新部署 |
| `Collection id is invalid because it contains /` | `collection_path` 多了尾巴 `/` | 寫 `wordcloud_words`，不要寫 `wordcloud_words/` |
| `firebase login` 在對話中卡住 | 互動式登入無法在 Codex 對話內完成 | 開 cmd/PowerShell 手動跑 `npx firebase-tools login` |
| Codex 重開前看不到 Firebase 工具 | MCP server 只會在啟動時載入 | 改完 `~/.codex/config.toml` 後，要完整重開 Codex Desktop |
| Windows 跑 `npx` 出問題 | PowerShell 可能先執行到 `npx.ps1` | 把 MCP 設定的 `command` 改成 `npx.cmd` |
| `codex --version` 顯示 `Access is denied` | Windows / Desktop 環境的 CLI 權限問題 | 不影響 Codex Desktop 實測；改走 GUI 或手動編輯 `~/.codex/config.toml` |
| `firestore_add_document` 說 document 格式 invalid | document 格式包錯 | 不要把 Firestore document 包成 JSON 字串；要傳 document 物件格式，例如 `fields.message.stringValue` |
